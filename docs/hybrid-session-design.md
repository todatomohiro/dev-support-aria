# ハイブリッドセッション設計書

> メイン会話（日常チャット）+ テーマ別ノート（専門会話）のハイブリッド方式

## 概要

既存の1対1メイン会話を維持しつつ、ユーザーが任意にテーマ別セッションを作成して専門的な会話を行える機能。
フルマルチセッション（Gemini/ChatGPT 型）ではなく、キャラクター型アプリの世界観を活かしたハイブリッド方式を採用する。

## 設計方針

- メイン会話は現状のまま（既存体験を壊さない）
- テーマ別セッションはユーザーが任意に作成・削除
- キャラクターは1人（メインもテーマ別も同じ AI キャラクター）
- テーマ別で話した内容は記憶を通じてメインに自然に反映される

## UI デザイン

> モックアップ: `docs/mockups/hybrid-session-responsive.html`（ブラウザで開いて確認可能）

### デザイン方針: PC = サイドバー方式 / スマホ = ボトムナビ方式

PC とスマホでナビゲーション方式を切り替えるレスポンシブ設計。

- **アプリタイトル（「AI Assistant」等）はヘッダーに表示しない** — キャラクター表示でアプリを識別できるため不要
- **ヘッダーには現在のセッション名を表示** — 「メイン会話」「旅行計画」等。ユーザーが今どのチャットにいるか一目でわかる

### PC 版: サイドバー方式（格納可能）

```
┌──────────────────────────────────────────────────────┐
│ ☰ │ 💬 メイン会話               [Live]              │
│───┼──────────────────────────────────────────────────│
│ 💬│ メイン会話    │            │                     │
│ ✈️│ 旅行計画     │  Live2D    │   チャット           │
│ 📚│ 英語学習     │  キャラ    │   メッセージ         │
│ 🍳│ 料理レシピ   │  クター    │   エリア             │
│ 💼│ 仕事の相談   │            │                     │
│───│──────────────│            │ [メッセージ入力...]  │
│ 👥│ グループ     │            │                     │
│ ⚙️│ 設定        │            │                     │
└──────────────────────────────────────────────────────┘
     サイドバー        Live2D       チャットエリア
    (260px, 格納可)   (280px→340px)   (残り全幅)
```

- **☰ トグルボタン**: メインエリアのヘッダー左端に配置。クリックでサイドバーを格納/展開
- **サイドバー格納時**: 260px 分がメインエリアに還元され、Live2D エリア（280px→340px）とチャットエリアが拡大
- **サイドバー内容**: メイン会話（常に最上部）→ テーマ別ノート一覧 → グループチャット → 設定
- **テーマ一覧への導線**: 「📝 テーマ別ノート」ラベルの横に `[一覧へ]` ボタンを配置。クリックでメインエリアにテーマカードグリッドを表示
- **テーマ一覧画面**: サイドバーはそのまま、メインエリアにテーマカードをグリッド表示

### スマホ版: ボトムナビ方式

```
┌─────────────────────┐
│ メイン会話       ⚙️  │  ← ヘッダー（現在のセッション名）
│─────────────────────│
│      Live2D         │
│    キャラクター      │
│─────────────────────│
│  チャット            │
│  メッセージ          │
│  エリア              │
│                     │
│  [メッセージ入力...] │
│─────────────────────│
│ 💬    📝    👥   ⚙️  │  ← ボトムナビ
│ チャット ノート グループ 設定│
└─────────────────────┘
```

- **ボトムナビ**: チャット / ノート / グループ / 設定 の4タブ
- **ヘッダー**: 現在のセッション名を表示（メイン会話 / テーマ名）
- **テーマチャット画面**: ヘッダーに「←」戻るボタン + テーマ名を表示。ボトムナビは維持
- 既存のグループチャット（`/groups`）と同じナビゲーションパターンで統一感を確保

## 記憶モデル: テーマ別 → メイン記憶への反映

### 基本方針

テーマ別セッションで話した内容は **永久記憶（事実）レベル** でメイン記憶に反映する。
会話の全文は反映しない（メインのコンテキストを汚染しないため）。

### 反映の仕組み（既存の3層記憶をそのまま活用）

```
テーマ別「旅行計画」セッション
  ↓ 会話中
  短期記憶: テーマ別セッション内で完結（メインに影響なし）
  中期記憶: AgentCore Memory に自動蓄積（メインから検索可能）
  ↓ セッション終了 or 一定時間経過
  extractFacts → 「来月京都旅行を計画中」を PERMANENT_FACTS に追加
  ↓
メイン会話
  AI「そういえば、京都旅行の計画どう？」 ← 自然に覚えている
```

### 各記憶層の扱い

| 記憶層 | テーマ別での扱い | メインへの反映 |
|--------|-----------------|---------------|
| 短期記憶（ローリング要約） | テーマ別セッション内で独立管理 | 反映しない |
| 中期記憶（AgentCore Memory） | テーマ別の会話も `/memory/events` に送信 | 自動的にメインから検索可能 |
| 永久記憶（PERMANENT_FACTS） | テーマ別セッション終了時に `extractFacts` 実行 | メインと共有（同じ PERMANENT_FACTS） |

### 理由: なぜ全文反映ではなく事実レベルか

- **全文反映の問題**: テーマ別で100ターン会話 → メインのコンテキストがテーマで埋まり、普段の雑談品質が下がる
- **反映しない場合の問題**: キャラクターがテーマ別で話したことを知らない → 記憶喪失に見える（ユーザー体験の低下）
- **事実レベル反映**: キャラクターとしての一貫性を保ちつつ、テーマ別の詳細がメインを汚染しないバランス

## DynamoDB 設計

### テーマ別セッションレコード

```
PK: USER#{userId}
SK: THEME_SESSION#{themeId}
themeName: "旅行計画"
createdAt: "2026-03-03T10:00:00Z"
updatedAt: "2026-03-03T12:00:00Z"
```

### テーマ別セッションのメッセージ

既存の MSG# と同じ構造。PK でテーマセッションを識別:

```
PK: USER#{userId}#THEME#{themeId}
SK: MSG#{timestamp}#user
role: "user"
content: "京都で行きたい場所をリストアップして"
createdAt: "2026-03-03T10:00:00Z"
ttlExpiry: （テーマ別セッションの TTL ポリシーに従う）
```

### テーマ別セッションの要約

```
PK: USER#{userId}
SK: THEME_SUMMARY#{themeId}
summary: "京都旅行を計画中。清水寺、伏見稲荷、嵐山を候補に..."
updatedAt: "2026-03-03T12:00:00Z"
```

## LLM Lambda の拡張

### テーマ別セッションのプロンプト構築

テーマ別セッションでは、既存の system prompt に加えてテーマコンテキストを注入:

```
<permanent_profile>（共有：メインと同じ永久記憶）</permanent_profile>
<user_context>（共有：AgentCore Memory）</user_context>
<theme_context>
  テーマ: 旅行計画
  このセッションでは「旅行計画」について会話しています。
  テーマに関連する回答を心がけてください。
</theme_context>
<current_session_summary>（テーマ別セッション独自の要約）</current_session_summary>
```

### エンドポイント

既存の `/llm/chat` を拡張。リクエストに `themeId` を追加:

```json
{
  "message": "京都で行きたい場所をリストアップして",
  "sessionId": "theme-session-uuid",
  "themeId": "旅行計画"
}
```

`themeId` がある場合はテーマ別セッションとして処理し、ない場合は既存のメイン会話として処理する。

## 必要な開発項目

### バックエンド（Lambda）

| 項目 | 説明 |
|------|------|
| テーマ別セッション CRUD | テーマ作成・一覧取得・削除の Lambda |
| `/llm/chat` 拡張 | `themeId` パラメータ対応、テーマコンテキスト注入 |
| `extractFacts` 拡張 | テーマ別セッション終了時にも永久記憶を抽出 |
| `summarize` 拡張 | テーマ別セッション内でも要約を生成 |

### フロントエンド

| コンポーネント | 説明 |
|---------------|------|
| `AppLayout` | レスポンシブレイアウト（PC: サイドバー / スマホ: ボトムナビ切り替え） |
| `Sidebar` | PC 版サイドバー（セッション一覧、☰ トグルで格納/展開） |
| `BottomNav` | スマホ版ボトムナビ（チャット / ノート / グループ / 設定） |
| `ThemeList` | テーマ別ノート一覧画面（PC: カードグリッド / スマホ: カードリスト） |
| `ThemeChat` | テーマ別チャット画面（メインの ChatUI をベースに拡張） |
| `CreateThemeModal` | テーマ名入力モーダル |

### サービス・ストア

| 項目 | 説明 |
|------|------|
| `themeService` | テーマ別セッション管理（CRUD + メッセージ送受信） |
| `themeStore`（Zustand） | テーマ一覧、アクティブテーマ、メッセージ状態 |

### ルーティング

| ルート | 画面 |
|--------|------|
| `/` | メイン会話（既存） |
| `/themes` | テーマ別ノート一覧 |
| `/themes/:themeId` | テーマ別チャット |

## 段階的な実装アプローチ

### Phase 1: テーマ別セッションの基本機能

- テーマ作成・一覧・削除
- テーマ別チャット画面（メインの ChatUI ベース）
- DynamoDB にテーマ別メッセージ保存
- 短期記憶はテーマ内で独立管理

### Phase 2: 記憶の連携

- テーマ別セッションの会話を AgentCore Memory に送信
- テーマ別セッション終了時に extractFacts でメイン永久記憶に反映
- テーマ別セッションのローリング要約

### Phase 3: UX の洗練

- テーマ別ノートの検索機能
- テーマの自動提案（メイン会話の内容からテーマを提案）
- テーマ別セッションの要約をメイン会話で参照可能にする
