プライベートモード + 選択的記憶

「AIが何を覚えるか」をユーザーがコントロール。お客様情報や会議内容がAIの人格記憶を汚染しないよう、
プライベートモードでは学習を停止し、本当に覚えてほしいことだけを明示的に記憶させます。

🟢 通常モード（学習ON）

  • ✓ 会話から自動で事実を抽出・記憶
  • ✓ 中期記憶（AgentCore）に保存
  • ✓ セッション終了時に永久記憶を更新
  • ✓ 過去の会話を踏まえた応答
適切な用途: 日常会話、個人的な相談、好みの共有

🔒 プライベートモード（学習OFF）

  • ✗ 会話内容を記憶しない
  • ✗ 中期記憶に保存しない
  • ✗ 永久記憶を更新しない
  • ✓ 既存の記憶は参照可能（読み取り専用）
  • ✓ 「覚えて」ボタンで選択的に記憶可能
適切な用途: 会議メモ、お客様情報、機密相談、一時的な調べ物

「覚えて」ボタン — 選択的記憶の操作方法

プライベートモードでも通常モードでも使える。AIの回答やまとめに対して「これは覚えて」と選択的に指示。

パターン1: ボタンで記憶

B社案件はPhase分割で進める方針になりました。
Phase1の予算は1500万以内です。
AIの回答の下に表示:
🧠 覚えて: 「B社はPhase分割方針」
🧠 覚えて: 「Phase1予算は1500万以内」
・AIが回答内容から記憶候補を自動提案
・クリックで永久記憶に保存
・もう一度クリックで取り消し

パターン2: 発話で記憶指示

B社案件ではAWSを使うことになった。これ覚えておいて。
覚えておきますね！
🧠 記憶しました: 「B社案件はAWSを使用」
・「覚えて」「記憶して」「メモして」などのキーワードで発動
・プライベートモードでも有効
・AIが内容を要約して永久記憶に保存

パターン3: 記憶の確認・削除

B社について覚えていることを教えて
B社について覚えていることです：

🧠 B社案件はPhase分割で進める方針 ✕ 忘れる
🧠 B社案件はAWSを使用 ✕ 忘れる
🧠 Phase1予算は1500万以内 ✕ 忘れる
・「覚えていること教えて」「B社の記憶を消して」で管理
・個別に「忘れる」で削除可能
・設定画面からも一覧確認・編集可能

設計 — プライベートモード + 選択的記憶

バックエンド変更点

会話フロー（Lambda /llm/chat）:

  1. リクエスト受信: { message, themeId, isPrivate, ... }
                                        ↓
  2. LLM 応答生成（通常通り — 既存記憶は参照可能）
                                        ↓
  3. 記憶処理の分岐:
     ┌─ isPrivate === false（通常モード）
     │   ├→ メッセージ保存（DynamoDB）           ✅ 実行
     │   ├→ 中期記憶保存（AgentCore Memory）     ✅ 実行
     │   ├→ ローリング要約（summarize）           ✅ 実行
     │   └→ セッション終了時: extractFacts        ✅ 実行
     │
     └─ isPrivate === true（プライベートモード）
         ├→ メッセージ保存（DynamoDB）            ✅ 実行（履歴は残す）
         ├→ 中期記憶保存（AgentCore Memory）      🚫 スキップ
         ├→ ローリング要約（summarize）            🚫 スキップ
         └→ セッション終了時: extractFacts         🚫 スキップ

選択的記憶（「覚えて」ボタン / 発話）:
  → POST /memory/remember { text: "B社案件はAWS使用", sourceThemeId }
  → DynamoDB PERMANENT_FACTS に直接追加
  → プライベートモードでも実行可能

DynamoDB スキーマ変更

変更対象変更内容
THEME_SESSION#{themeId} + isPrivate: boolean 属性追加
PERMANENT_FACTS + source: "auto" | "manual"（自動抽出 vs 手動記憶を区別）

API 変更点

エンドポイント変更

DELETE /memory/remember/{id} (新規)手動記憶の個別削除（「忘れて」）

実装の影響範囲（小さい）

核心的な変更はシンプル:
Lambda /llm/chat の記憶処理部分に if (!isPrivate) の分岐を1箇所追加するだけ。
既存の記憶システム（extractFacts, summarize, AgentCore）には変更不要。
フロントエンドは UI 表示の追加が中心。

既存機能への影響: なし（isPrivate のデフォルトは false → 従来通り学習ON）
