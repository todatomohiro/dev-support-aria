設計 — データモデル + API

既存のDynamoDBスキーマを拡張してTODO + トピック管理を実装。

TODO → 会議メモ連携フロー

会議終了 AIが自動要約 +
アクションアイテム抽出
→
TODO候補表示 ワンクリック or
AI指示でTODO化
→
TODO保存 担当者・期日・優先度
+ ソーストピックID
→
TODOビュー 横断一覧・フィルタ
ソースにジャンプ

DynamoDB スキーマ

PKSK属性説明
USER#{userId} TODO#{todoId} text, assignee, dueDate, priority, status, sourceThemeId, sourceType TODOアイテム
USER#{userId} THEME_SESSION#{themeId} + isPinned, isArchived, themeType(chat|meeting), todoCount トピック（既存拡張）

GSI 活用

インデックス用途
GSI1: userId + dueDate期限順のTODO一覧取得
GSI2: userId + status#priorityステータス別・優先度順

API エンドポイント（新規）

エンドポイントメソッド説明
/todosGETTODO一覧（フィルタ: status, dueDate, assignee）
/todosPOSTTODO作成（手動 or AI経由）
/todos/{todoId}PATCHTODO更新（ステータス、優先度、期日変更）
/todos/{todoId}DELETETODO削除
/themes/{themeId}PATCHトピック更新（+isPinned, +isArchived）
/themes/searchGETトピック全文検索（OpenSearch or DDB Scan）

LLM Tool Use 拡張

ツール名説明
create_todoTODOを作成（text, assignee, dueDate, priority）
list_todosTODO一覧取得（フィルタ条件指定可）
update_todoTODOのステータス・属性変更
search_topics過去トピック検索（会議メモ横断参照用）

Phase内容規模感
