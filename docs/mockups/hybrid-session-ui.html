A. サイドバー方式

左にセッション一覧、右にチャット。Gemini / ChatGPT に近いレイアウト。
メイン会話とテーマ別ノートを同じサイドバーで管理する。

メリット・セッション間の切り替えが速い
・一覧性が高く、全テーマを俯瞰できる
・PC の広い画面を有効活用
デメリット・モバイルではサイドバーが邪魔になる
・Gemini/ChatGPT と似すぎる印象
・キャラクターの表示スペースが圧迫される

B. タブ方式

ブラウザタブのように、ヘッダー下にセッションタブを並べる方式。
メイン会話は常に最左に固定、テーマ別ノートは右に追加される。

メリット・直感的で分かりやすい
・画面の横幅を有効活用
・メイン会話が常に目立つ位置にある
デメリット・テーマが増えるとタブが溢れる
・モバイルではタブが狭くなる
・キャラクター表示スペースは変わらない

C. ボトムナビ方式

モバイルアプリに最適。下部ナビゲーションでメイン会話とテーマ別ノートを切り替える。
既存のグループチャット（/groups）と同じナビゲーションパターン。

メリット・モバイルファーストで使いやすい
・メイン会話の体験を全く変えない
・既存のグループチャットと統一感がある
・キャラクターとの「メイン」感が維持される
デメリット・PC では下部ナビが不自然
・画面遷移が発生する（タブ切り替えより遅い）

D. フローティングボタン方式

メイン画面にフローティングボタンを配置。タップでテーマ一覧メニューが開く。
メイン画面のレイアウトを一切変えずに機能を追加できる。

メリット・既存の UI を全く変更しない
・実装が最も軽い
・モバイル・PC 両対応
デメリット・テーマが増えるとメニューが長くなる
・ボタンがチャットに重なる可能性
・気づきにくい（ディスカバラビリティが低い）
