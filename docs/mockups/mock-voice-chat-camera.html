設計メモ

カメラ映像送信機能の技術設計

データフロー

┌─ 音声会話 + カメラ映像送信 ────────────────────────────┐ │ │ │ [カメラ] → useCamera hook │ │ ↓ video stream (PiP プレビュー表示) │ │ ↓ captureFrame() → JPEG base64 (512px, 0.7q) │ │ │ │ [マイク] → Web Speech API (STT) │ │ ↓ テキスト確定 │ │ ↓ + カメラ ON なら captureFrame() を添付 │ │ │ │ chatController.sendMessage(text, imageBase64, true) │ │ ↓ │ │ Lambda /llm/chat │ │ ↓ detectImageFormat() → Bedrock ImageBlock │ │ ↓ Bedrock Converse API (テキスト + 画像) │ │ ↓ WebSocket chat_delta → TTS → スピーカー │ │ │ │ コスト: 画像 ~1,000-2,000 tokens/枚 (512px JPEG) │ │ レイテンシ: +100-200ms (base64エンコード+転送) │ └───────────────────────────────────────────────────────┘

UI 設計ポイント

  • カメラ ON/OFF: コントロールバーにカメラアイコンボタンを追加
  • PiP プレビュー: 右上に小窓 (110x148px) でカメラ映像を表示
  • 前面/背面切替: PiP 右下の回転アイコンで facingMode を切替
  • カメラ状態表示: PiP 左上に赤点滅 + 「前面」/「背面」ラベル
  • キャプチャタイミング: STT 確定時に自動キャプチャ（毎ターン1枚）
  • PiP 位置: Live2D キャラクターと重ならないよう右上に配置

前面/背面カメラ切替

  • useCamera の facingMode を 'user' / 'environment' で切替
  • 切替時は一旦 stop() → start(newFacingMode)
  • PC (Webカメラのみ) の場合は切替ボタンを非表示
  • navigator.mediaDevices.enumerateDevices() で複数カメラの有無を検出
  • デバイスが1台のみの場合は切替ボタンを disabled に

既存インフラとの整合性

  • useCamera hook: 既に captureFrame() (JPEG 512px base64) を実装済み
  • chatController.sendMessage: 第2引数 imageBase64 対応済み
  • llmClient: REST body に imageBase64 を含めて送信済み
  • Lambda: detectImageFormat() + toConverseMessages() で Bedrock に画像送信済み
  • 新規実装不要のレイヤー: llmClient, Lambda, Bedrock 連携はすべて既存コードで対応可能

注意事項

  • getUserMedia と SpeechSynthesis: useCamera の getUserMedia({video: true}) は video のみなので audio activation token への影響は少ないが、要検証
  • トークンコスト: 512px JPEG で約 1,000〜2,000 tokens/枚。10ターン会話で +10,000〜20,000 tokens
  • プライバシー: カメラ映像は base64 で Lambda に送信 → Bedrock で処理。DynamoDB にはテキストのみ保存（画像は保存しない）
  • バッテリー: カメラ常時 ON はバッテリー消費大。省電力のため OFF がデフォルト
  • iOS Capacitor: WKWebView でのカメラアクセスは既に useCamera で対応済み

実装ステップ

  • Step 1: useCamera に facingMode 切替機能を追加
  • Step 2: VoiceChatScreen にカメラ ON/OFF ボタン + PiP プレビューを追加
  • Step 3: handleSpeechResult でカメラ ON 時に captureFrame() → sendMessage(text, imageBase64)
  • Step 4: PiP 内に前面/背面切替ボタンを追加（複数カメラ検出時のみ表示）
  • Step 5: SpeechSynthesis との競合テスト（video-only getUserMedia の影響確認）
